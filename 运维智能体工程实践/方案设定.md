# 方案设定

## skill 使用策略

### Skill Router

skill roter 需要我们自己写中间件
**有以下两种：**

```python
# skill registry
skills = load_all_skills()

# embedding router
def select_skills(query):
    q_emb = embed(query)
    return similarity_search(q_emb, skills, top_k=5)

# agent runtime
selected = select_skills(user_input)

agent.run(
    tools=selected
)
```

还有一种：

```python
# 第一层：domain筛选
domain = domain_router(query)

# 第二层：只在domain内选skill
skills = load_domain_skills(domain)

selected = skill_router(query, skills)

agent.run(tools=selected)

```

这种先对 skills 进行分组，然后把分组的描述进行 embedding，最后根据 query 的 embedding 和分组描述的 embedding 进行相似度计算，选择最相关的分组。
然后再对分组内的 skill 进行 embedding，最后根据 query 的 embedding 和 skill 的 embedding 进行相似度计算，选择最相关的 skill。

当前阶段我们可以先用第一种，后面 skill 多了再考虑第二种。

补充一句，claude code 的渐进式加载是：
**先只加载 skill 元信息（catalog/index），再按需加载完整 skill 定义。元信息体量非常小**
