# 接入飞书

本文参考 OpenClaw 官方文档整理，并结合实际接入流程做了简化，目标是让你按步骤操作就能把飞书机器人接进 OpenClaw。

官方说明：

- [OpenClaw Feishu 文档](https://docs.openclaw.ai/channels/feishu)

---

# 一、先理解这件事是在做什么

简单说，就是：

1. 先在飞书开放平台创建一个机器人应用
2. 拿到 `App ID` 和 `App Secret`
3. 把这两个值配置到 OpenClaw
4. 启动 OpenClaw Gateway
5. 在飞书里给机器人发消息测试

OpenClaw 官方推荐使用飞书的 **长连接（WebSocket）模式** 收消息。  
这意味着：

- 一般不需要自己额外暴露公网回调地址
- 配置起来比 webhook 更省事
- 大多数场景直接按本文步骤来就够了

---

# 二、确认 OpenClaw 已安装完成

如果你已经能正常执行下面这些命令，就可以继续：

```bash
openclaw --help
openclaw gateway --help
openclaw channels add
```

根据官方文档，当前版本的 OpenClaw 一般已经内置 Feishu 插件，不需要单独安装。  
如果你使用的是旧版本，官方给出的安装命令是：

```bash
openclaw plugins install @openclaw/feishu
```

---

# 三、在飞书开放平台创建应用

## 1. 打开平台

打开飞书开放平台：

- 国内飞书：<https://open.feishu.cn/app>
- 国际版 Lark：<https://open.larksuite.com/app>

如果你是 **Lark 国际版租户**，后面在 OpenClaw 配置里需要加上：

```json
{
  "channels": {
    "feishu": {
      "domain": "lark"
    }
  }
}
```

## 2. 创建企业自建应用

进入平台后：

1. 新建应用
2. 选择企业自建应用
3. 填写应用名称、描述、图标
4. 创建完成

建议名称取一个容易识别的名字，比如：

- `OpenClaw AI 助手`
- `研发助手`
- `个人 AI 助理`

## 3. 复制凭证

在应用后台的“凭证与基础信息”页面，记下这两个值：

- `App ID`，格式通常像 `cli_xxx`
- `App Secret`

这两个值后面要填到 OpenClaw 配置里。

注意：

- `App Secret` 不要泄露
- 如果怀疑泄露，去飞书后台重新生成

---

# 四、给应用开权限

根据官方文档，最省事的方式是在“权限管理”里使用 **批量导入权限**。

可以导入下面这份权限配置：

```json
{
  "scopes": {
    "tenant": [
      "aily:file:read",
      "aily:file:write",
      "application:application.app_message_stats.overview:readonly",
      "application:application:self_manage",
      "application:bot.menu:write",
      "cardkit:card:read",
      "cardkit:card:write",
      "contact:user.employee_id:readonly",
      "corehr:file:download",
      "event:ip_list",
      "im:chat.access_event.bot_p2p_chat:read",
      "im:chat.members:bot_access",
      "im:message",
      "im:message.group_at_msg:readonly",
      "im:message.p2p_msg:readonly",
      "im:message:readonly",
      "im:message:send_as_bot",
      "im:resource"
    ],
    "user": [
      "aily:file:read",
      "aily:file:write",
      "im:chat.access_event.bot_p2p_chat:read"
    ]
  }
}
```

如果你不想批量导入，也至少要保证和消息接收、消息发送、资源读取相关的权限已经开通，否则机器人可能收得到消息却发不出去。

---

# 五、开启机器人能力

在飞书应用后台中找到：

`应用能力` -> `机器人`

然后：

1. 开启机器人能力
2. 设置机器人名称
3. 保存配置

如果这一步没开，后面即使 OpenClaw 配好了，飞书里也找不到可以聊天的 bot。

---

# 六、配置事件订阅

这一步非常关键。

官方建议使用：

- **长连接接收事件（WebSocket）**

然后订阅事件：

- `im.message.receive_v1`

建议按这个顺序来做：

1. 先在 OpenClaw 里把飞书渠道加好
2. 再启动 OpenClaw Gateway
3. 最后回到飞书后台保存事件订阅

原因是官方文档特别提醒：  
如果 Gateway 没启动，飞书那边的长连接配置有时可能保存不成功。

---

# 七、在 OpenClaw 中添加飞书渠道

官方推荐的方式有两种。

## 方法 1：初始化向导

如果你是刚安装完 OpenClaw，可以直接运行：

```bash
openclaw onboard
```

按提示选择 Feishu，然后填入：

- `App ID`
- `App Secret`

## 方法 2：单独添加渠道

如果你已经完成 OpenClaw 初始化，更常用的是：

```bash
openclaw channels add
```

执行后：

1. 选择 `Feishu`
2. 输入 `App ID`
3. 输入 `App Secret`

这是最适合大多数人的做法。

---

# 八、也可以直接写配置文件

如果你习惯手动维护配置，也可以直接编辑：

```bash
~/.openclaw/openclaw.json
```

一个最常见、最适合新手的飞书配置大概如下：

```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "dmPolicy": "pairing",
      "accounts": {
        "main": {
          "appId": "cli_xxx",
          "appSecret": "你的_app_secret",
          "botName": "OpenClaw 助手"
        }
      }
    }
  }
}
```

字段说明：

| 字段 | 作用 |
| --- | --- |
| `enabled` | 是否启用飞书渠道 |
| `dmPolicy` | 私聊策略，推荐先用 `pairing` |
| `accounts.main.appId` | 飞书应用的 App ID |
| `accounts.main.appSecret` | 飞书应用的 App Secret |
| `accounts.main.botName` | 机器人显示名称 |

---

# 九、为什么推荐先用 pairing 模式

`dmPolicy: "pairing"` 的意思是：

- 别人第一次私聊机器人时，不能直接开始聊天
- OpenClaw 会给他一个配对码
- 只有你在本地批准后，对方才能正式使用

这样做的好处是更安全。  
否则只要别人能找到这个 bot，就可能直接和它对话。

常用命令：

```bash
openclaw pairing list feishu
openclaw pairing approve feishu <配对码>
```

比如看到有一个待审批配对码：

```bash
openclaw pairing approve feishu 493021
```

批准后，这个飞书用户就能正常使用机器人了。

# 十、启动 Gateway

配置完成后，启动 OpenClaw Gateway：

```bash
openclaw gateway
```

或者先查看状态：

```bash
openclaw gateway status
```

再配合日志观察是否正常：

```bash
openclaw logs --follow
```

如果日志里没有明显报错，一般说明连接已经起来了。

# 十一、在飞书里测试

## 1. 发布应用

飞书应用通常还需要在后台完成：

1. 创建版本
2. 提交发布
3. 等待审核/管理员批准

如果应用没有发布成功，机器人可能无法正常被使用。

## 2. 给机器人发消息

在飞书里找到你的 bot，发送一条消息，例如：

```text
hello
```

## 3. 处理 pairing

如果你启用了默认的 `pairing` 模式，第一次发消息时：

- 飞书侧会收到一个配对码
- 你需要在本地执行审批命令

```bash
openclaw pairing list feishu
openclaw pairing approve feishu <配对码>
```

审批通过后，再发一次消息，机器人就应该能正常回复。

---

# 十二、群聊里怎么用

OpenClaw 官方文档里提到，飞书群聊默认更适合使用 **@机器人再提问** 的方式。

比如可以这样理解：

- 私聊：默认按 pairing 控制是否允许使用
- 群聊：默认建议 `@机器人` 时才回复，避免打扰整个群

如果你要让某个群必须 `@` 才回复，可以这样写：

```json
{
  "channels": {
    "feishu": {
      "groups": {
        "oc_xxx": {
          "requireMention": true
        }
      }
    }
  }
}
```

如果你明确希望某个群里不需要 `@` 也能回复：

```json
{
  "channels": {
    "feishu": {
      "groups": {
        "oc_xxx": {
          "requireMention": false
        }
      }
    }
  }
}
```

其中：

- `oc_xxx` 是飞书群 ID

如果你只想允许指定群接入，可以再加白名单：

```json
{
  "channels": {
    "feishu": {
      "groupPolicy": "allowlist",
      "groupAllowFrom": ["oc_xxx", "oc_yyy"]
    }
  }
}
```

# 十三、常见问题

## 1. 机器人收不到消息

优先检查下面几项：

1. 飞书应用是否已经发布
2. 机器人能力是否已经开启
3. 事件订阅是否选择了“长连接”
4. 是否已经添加事件 `im.message.receive_v1`
5. `openclaw gateway` 是否正在运行

## 2. 能收到消息，但机器人不回复

常见原因：

1. 还没有批准 pairing
2. 权限没配全，导致发送消息失败
3. `App ID` 或 `App Secret` 填错
4. 日志里已经报错，但没有及时看见

建议直接看：

```bash
openclaw logs --follow
```

## 3. 群里不响应

先检查：

1. 当前群是否允许接入
2. 是否要求必须 `@机器人`
3. 群配置里 `chat_id` 是否写对

## 4. 国际版 Lark 无法正常工作

检查是否配置了：

```json
{
  "channels": {
    "feishu": {
      "domain": "lark"
    }
  }
}
```

# 十四、一个适合新手直接照抄的最小配置

如果你只想先跑通，可以先用这份：

```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "dmPolicy": "pairing",
      "accounts": {
        "main": {
          "appId": "cli_xxx",
          "appSecret": "xxx",
          "botName": "OpenClaw 助手"
        }
      }
    }
  }
}
```

然后执行：

```bash
openclaw gateway
```

再去飞书里给机器人发消息，看到配对码后审批：

```bash
openclaw pairing list feishu
openclaw pairing approve feishu <配对码>
```

这套流程跑通以后，再考虑做群聊白名单、多账号、国际版域名这些进阶配置。

---

# 十五、推荐的接入顺序

如果你想尽量少踩坑，推荐严格按下面顺序做：

1. 在飞书开放平台创建应用
2. 开启机器人能力
3. 配置权限
4. 在 OpenClaw 中添加 Feishu 渠道
5. 启动 `openclaw gateway`
6. 回到飞书后台配置长连接事件订阅
7. 发布应用
8. 在飞书里发送消息测试
9. 本地批准 pairing

按这个顺序，一般最稳。
