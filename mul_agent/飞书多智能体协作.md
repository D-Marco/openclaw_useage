# OpenClaw 飞书群聊三智能体协作部署指南

> 适用版本：v2026.1.6+
> 目标：在飞书群聊中部署 sys_arch、be_coder、fe_coder 三个智能体，支持 @ 触发与互相协作

---

## 一、整体架构

```
飞书群聊
   │
   ├── @sys_arch  → 系统架构师（负责技术选型、架构设计、任务拆解）
   │                  ↕ 可调用
   ├── @be_coder  → 后端工程师（负责服务端代码、API、数据库）
   │                  ↕ 可调用
   └── @fe_coder  → 前端工程师（负责页面、组件、交互逻辑）
```

**协作模式**：用户在群聊中 @ 对应智能体发起任务，智能体之间可按需互相调用完成复杂任务。

---

## 二、部署步骤

### 步骤 1：创建三个智能体

在服务器上执行：

```bash
openclaw agents add sys_arch
openclaw agents add be_coder
openclaw agents add fe_coder
```

执行后自动创建以下工作区目录：

- `~/.openclaw/workspace-sys_arch/`
- `~/.openclaw/workspace-be_coder/`
- `~/.openclaw/workspace-fe_coder/`

### 步骤 2：修改 openclaw.json

将 `agents` 部分替换为以下配置：

```json
"agents": {
  "defaults": {
    "model": {
      "primary": "vllm/Qwen3.5-27B"
    },
    "workspace": "/home/lane/.openclaw/workspace",
    "compaction": {
      "mode": "safeguard"
    }
  },
  "list": [
    {
      "id": "sys_arch",
      "name": "系统架构师",
      "mentionPatterns": ["@sys_arch", "@系统架构师"],
      "tools": {
        "profile": "minimal",
        "agentToAgent": {
          "enabled": true,
          "allow": ["be_coder", "fe_coder"]
        }
      }
    },
    {
      "id": "be_coder",
      "name": "后端工程师",
      "mentionPatterns": ["@be_coder", "@后端工程师"],
      "tools": {
        "profile": "coding",
        "agentToAgent": {
          "enabled": true,
          "allow": ["sys_arch", "fe_coder"]
        }
      }
    },
    {
      "id": "fe_coder",
      "name": "前端工程师",
      "mentionPatterns": ["@fe_coder", "@前端工程师"],
      "tools": {
        "profile": "coding",
        "agentToAgent": {
          "enabled": true,
          "allow": ["sys_arch", "be_coder"]
        }
      }
    }
  ],
  "bindings": [
    {
      "comment": "飞书群聊消息统一进入多智能体路由（由 mentionPatterns 决定具体 agent）",
      "agentId": "sys_arch",
      "match": {
        "channel": "feishu",
        "accountId": "main"
      }
    }
  ]
}
```

> **说明**：`bindings` 中配置 `sys_arch` 作为默认接收方，群聊中未 @ 任何人时由架构师兜底处理；具体 @ 触发由 `mentionPatterns` 优先匹配。

### 步骤 3：为每个智能体配置人格（SOUL.md）

```bash
cat > ~/.openclaw/workspace-sys_arch/SOUL.md << 'EOF'
你是一位资深系统架构师，擅长技术选型、架构设计和任务拆解。
收到需求时，先给出整体架构方案，再将前后端任务分别分配给 be_coder 和 fe_coder。
回答简洁清晰，使用 Markdown 格式输出架构图和任务清单。
EOF

cat > ~/.openclaw/workspace-be_coder/SOUL.md << 'EOF'
你是一位专业的后端工程师，擅长服务端开发、API 设计和数据库建模。
优先给出可运行的代码示例，语言默认使用 Python 或 Go，除非用户指定。
遇到需要前端配合的部分，主动说明接口规范。
EOF

cat > ~/.openclaw/workspace-fe_coder/SOUL.md << 'EOF'
你是一位专业的前端工程师，擅长 React/Vue 组件开发、页面布局和交互逻辑。
优先给出可运行的代码示例，默认使用 React + TypeScript。
遇到需要后端接口的部分，主动对齐接口规范。
EOF
```

### 步骤 4：重启服务

```bash
openclaw restart
# 或
sudo systemctl restart openclaw
```

### 步骤 5：验证配置

```bash
openclaw agents list --bindings
```

## 三、在飞书群聊中使用

| 使用方式 | 示例 | 处理智能体 |
|---------|------|----------|
| @ 架构师 | `@sys_arch 设计一个电商系统` | sys_arch |
| @ 后端 | `@be_coder 实现用户登录接口` | be_coder |
| @ 前端 | `@fe_coder 做一个商品列表页` | fe_coder |
| 不 @ 任何人 | `帮我看看这个需求` | sys_arch（兜底） |

**智能体协作示例**：

```
用户：@sys_arch 我要做一个博客系统，帮我规划一下

sys_arch：（输出架构方案后）→ 调用 be_coder 实现后端 API
                             → 调用 fe_coder 实现文章列表页
```

## 六、常见问题

### Q: @ 了智能体但没有响应？

**A**: 检查两点：
1. `mentionPatterns` 中的值要和飞书群里实际 @ 的机器人名称一致
2. 确认机器人已被加入该群聊，且有消息读取权限

### Q: 智能体之间无法互相调用？

**A**: 确认双方的 `agentToAgent.allow` 列表中都包含了对方的 `id`，且 `enabled: true`。

### Q: 三个智能体会共享对话历史吗？

**A**: 不会。每个智能体的会话独立存储在 `~/.openclaw/agents/<agentId>/sessions/`，互不干扰。
